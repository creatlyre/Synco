<objective>
Create dedicated Login and Register HTML pages with email/password forms and Google OAuth button, replacing the current direct-redirect behavior of GET /auth/login. Pages use Synco's glassmorphic dark theme with client-side validation and inline error handling.

Purpose: Users currently can only sign in via Google OAuth redirect. This adds a proper login/register UI with email/password authentication as an alternative, matching the branded Synco design.

Output: 2 new templates (login.html, register.html), modified routes + middleware, i18n keys, tests.

**1. Modify `app/auth/routes.py`:**

Replace the existing `GET /auth/login` handler. Instead of redirecting to Google OAuth, render a `login.html` template. Add a query param `?provider=google` that preserves the old redirect behavior (for Google OAuth button clicks):

```python
@router.get("/login")
async def login_page(request: Request, provider: str | None = None, db=Depends(get_db)):
    settings = Settings()
    if provider == "google":
        if is_supabase_auth_enabled(settings):
            auth_url = build_google_authorize_url(settings.GOOGLE_REDIRECT_URI)
            return RedirectResponse(url=auth_url)
        auth_url, _state = get_authorization_url()
        return RedirectResponse(url=auth_url)

    locale = resolve_locale(request)
    context = {"request": request, "locale": locale}
    inject_template_i18n(context, request)
    return templates.TemplateResponse("login.html", context)
```
