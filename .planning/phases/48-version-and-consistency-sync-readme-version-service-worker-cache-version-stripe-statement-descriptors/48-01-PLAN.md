<task type="auto">
  <name>Task 2: Add Stripe statement descriptor to checkout</name>
  <files>app/billing/service.py</files>
  <action>
1. In app/billing/service.py, in the `create_checkout_session` method, add `payment_intent_data` with statement descriptor for subscription mode. The `statement_descriptor` max is 22 chars, only alphanumeric, spaces, and hyphens.

   For subscription mode (which creates a subscription, not a one-time payment), Stripe uses the subscription's `payment_settings.statement_descriptor` instead. The simplest approach is to add it to the session:

   In the `stripe.checkout.Session.create()` call (line ~69), add the following parameter for subscription mode:
   ```python
   session = stripe.checkout.Session.create(
       mode=mode,
       customer=stripe_customer_id,
       line_items=[{"price": price_id, "quantity": 1}],
       success_url=success_url,
       cancel_url=cancel_url,
       metadata={"user_id": user_id, "plan": plan},
       **({"payment_intent_data": {"statement_descriptor": "DOBRYPLAN"}} if mode == "payment" else
          {"subscription_data": {"metadata": {"user_id": user_id, "plan": plan}}}),
   )
   ```

   Actually, simpler approach — add both subscription_data and payment_intent_data conditionally:
   ```python
   extra = {}
   if mode == "subscription":
       extra["subscription_data"] = {
           "metadata": {"user_id": user_id, "plan": plan},
       }
   else:
       extra["payment_intent_data"] = {
           "statement_descriptor": "DOBRYPLAN",
       }

   session = stripe.checkout.Session.create(
       mode=mode,
       customer=stripe_customer_id,
       line_items=[{"price": price_id, "quantity": 1}],
       success_url=success_url,
       cancel_url=cancel_url,
       metadata={"user_id": user_id, "plan": plan},
       **extra,
   )
   ```

   Note: For subscriptions, the statement descriptor is set on the Stripe product/price level or in the Stripe Dashboard → Settings → Public details → Statement descriptor. The code-level approach above handles one-time payments (self-hosted).
  </action>
