**1. Create** `app/shopping/views.py`:
```python
from fastapi import APIRouter, Depends, Request
from fastapi.responses import HTMLResponse
from fastapi.templating import Jinja2Templates
from app.auth.dependencies import get_current_user
from app.database.database import get_db
from app.i18n import inject_template_i18n, set_locale_cookie_if_param

router = APIRouter(tags=["shopping-views"])
templates = Jinja2Templates(directory="app/templates")

@router.get("/shopping", response_class=HTMLResponse)
async def shopping_page(request: Request, user=Depends(get_current_user), db=Depends(get_db)):
    context = inject_template_i18n(request, {"request": request, "user": user})
    response = templates.TemplateResponse(request=request, name="shopping.html", context=context)
    set_locale_cookie_if_param(response, request)
    return response
```

**2. Create** `app/templates/shopping.html` extending base.html with glass-morphism design:

The template must contain:
- `{% extends "base.html" %}` and `{% block content %}`
- Page title using `{{ t('shopping.title') }}`
- **Add item input**: text input + "Add" button. On submit, POST to `/api/shopping/items` with the item name. JavaScript `fetch`.
- **Multi-add input**: textarea + "Add All" button. On submit, POST to `/api/shopping/items/multi` with the text. Clear textarea after success.
- **Section-grouped item display**: For each section, render a collapsible section header with emoji + section name. Under each header, list items as rows with:
  - Item name text
  - Edit button (pencil icon) — inline edit with input field
  - Delete button (trash icon) — DELETE to `/api/shopping/items/{id}`
- **Uncategorized section**: Items without a section shown under "Inne / Other" with a "Pick section" dropdown per item. On selection, POST to `/api/shopping/items/{id}/learn` with the section_id and item_name.
- **Empty state**: glass card with message when no items exist
- All interactive operations use `fetch()` to the API and update DOM without full page reload (same pattern as budget pages)
- Use Tailwind glass-morphism classes matching existing app style: `glass`, `glass-btn-secondary`, `rounded-xl`, etc.
- Mobile-friendly: 44px min touch targets, responsive layout
