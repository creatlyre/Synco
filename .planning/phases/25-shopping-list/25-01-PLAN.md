```sql
-- Shopping sections (store aisles)
create table if not exists public.shopping_sections (
  id uuid primary key default gen_random_uuid(),
  calendar_id text not null references public.calendars(id) on delete cascade,
  name text not null,
  emoji text not null default '',
  sort_order int not null default 0,
  is_preset boolean not null default true,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),
  unique (calendar_id, name)
);
alter table public.shopping_sections disable row level security;

-- Shopping items
create table if not exists public.shopping_items (
  id uuid primary key default gen_random_uuid(),
  calendar_id text not null references public.calendars(id) on delete cascade,
  name text not null,
  section_id uuid references public.shopping_sections(id) on delete set null,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);
alter table public.shopping_items disable row level security;

-- Learned keyword mappings (user teaches the system)
create table if not exists public.shopping_keyword_overrides (
  id uuid primary key default gen_random_uuid(),
  calendar_id text not null references public.calendars(id) on delete cascade,
  keyword text not null,
  section_id uuid not null references public.shopping_sections(id) on delete cascade,
  created_at timestamptz not null default now(),
  unique (calendar_id, keyword)
);
alter table public.shopping_keyword_overrides disable row level security;
```

**1. Create** `app/shopping/service.py`:

```python
class ShoppingService:
    def __init__(self, repo: ShoppingRepository):
        self.repo = repo
```

Key methods:
- `ensure_sections(calendar_id)` — calls `repo.init_preset_sections(calendar_id)` if no sections exist
- `list_items(calendar_id)` → returns items grouped by section: `{"sections": [...], "items": [...], "uncategorized": [...]}`
- `add_item(calendar_id, name, section_id=None)` → if no section_id, auto-categorize via keyword matching → create item
- `add_multiple(calendar_id, text)` → split on commas and newlines, strip whitespace, filter empty, call `add_item` for each. Return list of created items with `{"items": [...], "uncategorized_names": [...]}`
- `update_item(item_id, data)` → delegate to repo
- `delete_item(item_id)` → delegate to repo
- `categorize_item(item_name)` → return section name or None. Logic:
  1. Load user overrides from DB, check first (user learned mappings take priority)
  2. Then check `SECTION_KEYWORDS` from keywords.json
  3. Use `_normalize()` (same diacritics-stripping as expense_service) for matching
  4. Match: for each keyword in section's list, check if normalized keyword is substring of normalized item name (or vice versa for words >= 3 chars)
- `learn_keyword(calendar_id, item_name, section_id)` → upsert override + update item's section_id. Normalized keyword stored.
- Section CRUD: `list_sections(calendar_id)`, `create_section(calendar_id, name, emoji, sort_order)`, `update_section(section_id, data)`, `delete_section(section_id)`
