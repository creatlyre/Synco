<task type="auto">
  <name>Task 2: Extract landing.html inline styles to CSS custom properties</name>
  <files>app/templates/landing.html</files>
  <action>
1. At the top of the existing `<style>` block in landing.html, define CSS custom properties on `:root`:
   ```css
   :root {
     --brand-bg: #0f0a2e;
     --brand-bg-rgb: 15, 10, 46;
     --brand-card-rgb: 26, 20, 68;
     --brand-primary: #6366f1;
     --brand-primary-rgb: 99, 102, 241;
     --brand-accent: #8b5cf6;
     --brand-accent-rgb: 139, 92, 246;
   }
   ```

2. Create named CSS classes for the most-repeated patterns:
   - `.bg-brand` → `background: var(--brand-bg);`
   - `.bg-brand-nav` → `background: rgba(var(--brand-bg-rgb), 0.85); backdrop-filter: blur(16px);`
   - `.bg-brand-card` → `background: linear-gradient(180deg, rgba(var(--brand-card-rgb), 0.5) 0%, rgba(var(--brand-bg-rgb), 0.4) 100%);`
   - `.bg-brand-card-strong` → same pattern with 0.7/0.8 opacity variant
   - `.glow-orb-primary` → `background: radial-gradient(circle, rgba(var(--brand-primary-rgb), 0.4), transparent 70%);`
   - `.glow-orb-accent` → `background: radial-gradient(circle, rgba(var(--brand-accent-rgb), 0.4), transparent 70%);`

3. Replace static inline `style=` attributes with the new classes:
   - `<body ... style="background: #0f0a2e;">` → `<body ... class="... bg-brand">`
   - `<nav ... style="background: rgba(15,10,46,0.85); backdrop-filter: blur(16px);">` → `<nav ... class="... bg-brand-nav">`
   - All `style="background: linear-gradient(180deg, rgba(26,20,68,0.5)..."` cards → add `bg-brand-card` class
   - Glow orbs → add `glow-orb-primary` / `glow-orb-accent` classes

4. Do NOT touch inline styles that use Jinja2 `{{ color }}` template variables — those are dynamic per-feature and must remain inline.

5. Leave `letter-spacing` and other one-off inline styles alone — they're not worth extracting.
  </action>
  <verify>
    <automated>python -c "t=open('app/templates/landing.html').read(); assert '--brand-bg' in t, 'custom properties missing'; assert 'bg-brand' in t, 'brand classes missing'; inline_count=t.count('style=\"background: #0f0a2e'); assert inline_count==0, f'still has {inline_count} hardcoded bg'; print('OK')"</automated>
  </verify>
  <done>Landing page uses CSS custom properties for brand colors. Static inline styles replaced with named classes. Dynamic Jinja2 styles preserved.</done>
</task>
